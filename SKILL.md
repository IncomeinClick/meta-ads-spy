# meta-ads-spy — ส่องแอดคู่แข่ง → ปั้นคอนเทนต์ยิงแอด

โบนัสจากคอร์ส **จ้างเอไอยิง Meta Ads**. หลักคิดของ media buyer มืออาชีพ:
**ก่อนคิดคอนเทนต์ยิงแอด ให้ไปดูก่อนว่าคู่แข่งตัวไหน "ยิงแล้วได้ผล" — แล้วต่อยอดให้ต่างแต่แข่งได้**
สัญญาณว่าแอดไหนได้ผล = **มันถูกยิงต่อเนื่องมานาน** (ไม่มีใครจ่ายเงินยิงแอดที่ขายไม่ออกเป็นเดือนๆ)

ทุกครั้งที่ผู้ใช้สั่งให้ "ทำคอนเทนต์ยิงแอด" — **อย่าเพิ่งเขียนคอนเทนต์ทันที** ให้เดินตาม protocol นี้ก่อนเสมอ: ส่องคู่แข่งก่อน → แล้วค่อยปั้นคอนเทนต์

---

## ก่อนเริ่ม: token (ครั้งเดียว)

สคริปต์ดึงข้อมูลจาก **Meta Ad Library API** ต้องใช้ token ของผู้ใช้เอง และบัญชีนั้นต้อง **ยืนยันตัวตนแล้ว** ที่ facebook.com/id (ครั้งเดียว รอ 1-3 วัน — มีบอกในอีเมลต้อนรับคอร์ส)

- token หาได้จาก: env `META_ADS_TOKEN` หรือไฟล์ `~/.config/meta-ads-spy/token`
- ถ้ายังไม่มี → ขอจากผู้ใช้ แล้วเซฟให้: `mkdir -p ~/.config/meta-ads-spy && printf '%s' '<TOKEN>' > ~/.config/meta-ads-spy/token`
- ใช้ **long-lived token** (อยู่ได้ ~60 วัน) จะได้ไม่ต้องต่อบ่อย
- ถ้าสคริปต์คืน `identity_not_confirmed` หรือ `token_expired` → บอกผู้ใช้ตามข้อความนั้น อย่าฝืนเดินต่อ

## Protocol (เดินตามลำดับ)

### 1) ถามข้อมูลก่อนค้น
ถามผู้ใช้ 2 อย่าง:
1. **ขายอะไร?** (สินค้า/บริการ) → เอามาตั้ง keyword สำหรับค้น
2. **มีแบรนด์คู่แข่งที่นึกออกไหม?** (ถ้ามี ดีมาก — ค้นด้วย page id ตรงกว่า keyword เยอะ)

ถ้าผู้ใช้ขายในต่างประเทศ ถาม country ด้วย (ดีฟอลต์ `TH`)

### 2) ค้น Ad Library
รันสคริปต์ (ปรับ path ตามที่ skill ถูกติดตั้ง):
```bash
python3 <SKILL_DIR>/scripts/adlib_search.py --query "<keyword>" --country TH
# หรือถ้ารู้เพจคู่แข่ง (สะอาดกว่า):
python3 <SKILL_DIR>/scripts/adlib_search.py --page-id <competitor_page_id> --country TH
```
สคริปต์คืน JSON: แอด active เรียงตาม **`days_running` มาก→น้อย** อยู่แล้ว (= winner ranking ดิบ)

> keyword ค้นจาก "ข้อความในแอด" จึงปนเปื้อนได้ — ถ้าผลมั่ว ลองค้นด้วยชื่อแบรนด์/เพจแทน

### 3) เลือก 5 แอด winner — **dedupe ด้วย "มุม" ไม่ใช่ "แบรนด์"**
จาก JSON:
1. โฟกัสตัวที่ `days_running` สูง (ยิงนาน = winner)
2. **อ่าน `body` แล้วจัดกลุ่มตาม "มุม/big idea"** (เช่น ลดราคา, before-after, social proof, แก้ pain, ความสะดวก, กลัวพลาด)
3. เลือก **5 มุมที่ต่างกัน** → แต่ละมุมหยิบตัวที่ยิงนานสุดเป็นตัวแทน
   - **แบรนด์เดียวโผล่ได้หลายครั้ง ถ้ามุมต่างกันจริง** (อย่าตัดทิ้งเพราะซ้ำแบรนด์ — แบรนด์ที่ยิงหลายมุมนานๆ = เจ้าตลาด ของดี)
   - ตัวที่ body/มุมซ้ำกัน → ยุบเหลือ 1 ตัวแทน
4. **Consensus tiebreak:** ถ้ามีมุมเข้าข่ายเกิน 5 → มุนที่ **หลายแบรนด์** ยิงนานเหมือนกัน สำคัญกว่ามุนที่เจ้าเดียวยิง (ทั้งตลาด validate แล้ว)
5. **ถ้ามุมต่างกันจริงมีไม่ถึง 5** → คืนเท่าที่มี (3-4) แล้วบอกตรงๆ อย่ายัด junk ให้ครบ
6. **ถ้า winner มีมากกว่า 5 มุม** → เสนอ 5 ก่อน แล้ว **ถามผู้ใช้ว่าอยากดูมุมเพิ่มอีกไหม**

### 4) รายงานผลให้ผู้ใช้
สำหรับแต่ละแอดที่เลือก บอก:
- **ขายอะไร** + **big idea/มุม** ที่เล่น
- **แบรนด์ไหน** + ยิงมานานกี่วัน (จาก `days_running`)
- ถ้าแบรนด์ไหนครอง 3+ มุมในลิสต์ → ชูเป็น insight: "แบรนด์ X ครองตลาด ยิงหลายมุมนาน = คู่แข่งหลักต้องจับตา"

จากนั้นเสนอ: **big idea ที่เราควรใช้** — ต่างจากคู่แข่งแต่ยังแข่งขันได้ (อธิบายว่าทำไม)

### 5) ขอ approve ก่อนร่างคอนเทนต์
**หยุดตรงนี้** — สรุป big idea ที่เสนอ แล้วถามผู้ใช้ว่า "เอา big idea นี้ไหม / อยากปรับมุมไหน" **อย่าเพิ่งร่างคอนเทนต์จนกว่าผู้ใช้จะ approve**

### 6) ร่างคอนเทนต์จริง (หลัง approve)
พอผู้ใช้ตกลง → ร่าง **คอนเทนต์/creative ยิงแอดจริงให้เลยในแชทเดียว** (เช่น 5 มุม: hook + body + CTA) ตาม big idea ที่เคาะ — นี่คือหัวใจ: ส่องเสร็จ → ได้ของใช้จริง ไม่ต้องสลับไปไหน

## Fallback (ห้ามตัน)
ถ้าสคริปต์คืน `count: 0` หรือผลน้อยเกินไป:
- ขยาย keyword ให้กว้างขึ้น / ลองคำ**ภาษาอังกฤษ** (บาง niche แอดเขียน eng)
- ลองค้นด้วยชื่อแบรนด์คู่แข่งแทน keyword
- บาง niche (อาหารเสริม/การเงิน/ลดน้ำหนัก) Meta จำกัดการแสดงผล → บอกผู้ใช้ตรงๆ แล้วขอชื่อคู่แข่งที่รู้มาช่วย
- อย่าแกล้งมีข้อมูลถ้าไม่มี — ถามเพิ่มดีกว่าเดา
